B. Add Logging to Silent Catches

P1

Replace .catch(() => {}) with .catch(e => logger.warn('context', {safeMessage: e})) in the files listed above. This surfaces hidden bugs via Sentry breadcrumbs and the logDump ring buffer without changing error handling behavior.

C. Sentry Spans for TanStack Query

P2

No API call tracing exists. A global wrapper around TanStack Query's queryFn can emit Sentry breadcrumbs/spans for every API call, providing per-endpoint latency percentiles without touching individual queries.

This can be implemented via QueryClient defaults or a custom queryFn wrapper that starts/finishes a Sentry span around the actual fetch.

D. Network Error Rate Tracking

P2

Sentry currently ignores AbortError and Network request failed entirely. Network reliability is unmeasured. Options:

  • Capture network errors as breadcrumbs (filter from issues but still count them)
  • Send lightweight metrics events from the logger transport on network errors

E. Feed Scroll Performance

P3

The most latency-sensitive UX path has zero instrumentation. Sentry auto-perf captures app start and frozen frames, but not per-feed-item render timing or scroll jank correlation. Use Sentry manual spans or lightweight custom metrics.

F. Test Coverage Reporting

P2

Jest runs in CI but coverage is not tracked. The test-coverage script exists (jest --coverage) but is not wired into any workflow. Add to lint.yml and post coverage diff comments on PRs.

G. Dependency Vulnerability Scanning

P1

No scanning exists. Add pnpm audit --audit-level=high as a step in lint.yml or enable GitHub Dependabot.

Strategic Decision: OTA Strategy

Option A: Custom Update Service (Rehosted)

  • Replace updates.bsky.app with Blacksky URL
  • Replace bundleUpdate.sh upload endpoint
  • Build or deploy equivalent update server
  • Rename DENIS_API_KEY to BLACKSKY_UPDATES_API_KEY
  • Replace PR QR generator URL (bsky-qr.vercel.app)
  • Replace bluesky:// deep link in PR comments

Option B: Expo EAS Update (Recommended)

  • Replace custom bundleUpdate.sh with eas update
  • Configure EAS project/channel/branch mapping
  • Remove custom upload secret and endpoint
  • Adjust app.config.js updates config
  • Rework PR OTA comments
  • Simpler setup if no update server exists

Recommendation: If Blacksky does not already operate a compatible update service, use Expo EAS Update. The current custom service is tightly coupled to Bluesky infrastructure.

Code Signing

Current config references ./code-signing/certificate.pem with algorithm rsa-v1_5-sha256. If using custom update service, confirm existing certificate is appropriate or regenerate. If using EAS Update, follow Expo's code signing setup.
