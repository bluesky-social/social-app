# Contributing to K4M2A

Thank you for your interest in contributing.

The goal of K4M2A is to build digital infrastructure for seekers, teachers, philosophical communities, and people working toward inner transformation, wisdom, and human development.

This project is not trying to maximize engagement, addiction, outrage, or attention capture. We are building technology for reflection, learning, community, and awakening.

## Current Focus

We are currently adapting the Bluesky social app into K4M2A.

The main work includes:

- Replacing Bluesky-specific branding with K4M2A/CoSeeker branding
- Making search and discovery relevant to K4M2A users
- Improving anti-spam and moderation flows
- Improving user identity, names, profiles, and onboarding
- Creating safer spaces for seekers and communities
- Improving documentation so new contributors can participate easily
- Solving issues and Feature Request from the users

## Who Can Contribute

You do not need to be an expert to contribute.

We welcome help from:

- React Native developers
- TypeScript developers
- AT Protocol / Bluesky ecosystem developers
- UI/UX designers
- Product thinkers
- Documentation writers
- Testers
- Community moderators
- People from spiritual, philosophical, educational, or human-development communities

## Good First Contributions

The best place to start is with issues labeled:

- `good first issue`
- `help wanted`
- `documentation`
- `frontend`
- `branding`
- `testing`

Good first tasks may include:

- Fixing text or UI labels
- Replacing old Bluesky references
- Improving documentation
- Testing flows and reporting bugs
- Improving profile or display-name behavior
- Helping clean up onboarding and settings screens

## Contributor Missions

Some first contributions do not require code.

CoSeeker needs contributors who can help with product thinking, UX review, community design, safety, documentation, and mission alignment.

Look for issues labeled:

- `contributor-mission`
- `good first issue`
- `no-code`
- `product-thinking`

## How to Contribute

1. Find an issue you want to work on.
2. Comment on the issue:  
   `I would like to work on this.`
3. Wait for a maintainer to confirm or assign the issue.
4. Fork the repository.
5. Create a new branch.
6. Make a small, focused change.
7. Open a pull request.
8. Clearly explain what you changed and why.

## Pull Request Guidelines

Please keep pull requests small and focused.

A good pull request should include:

- A clear title
- A short explanation of the problem
- A short explanation of the solution
- Screenshots or screen recordings for UI changes, when useful
- Testing notes, if applicable

Avoid combining unrelated changes into one pull request.

## Code and Product Principles

When contributing, please keep these principles in mind:

- Build for human development, not attention capture.
- Prefer clarity over cleverness.
- Avoid addictive engagement mechanics.
- Respect user safety, privacy, and dignity.
- Design for communities, not just individuals.
- Make the app easier for sincere seekers and teachers to use.
- Keep the project understandable for future contributors.

## Communication

Use GitHub Issues for specific tasks and bug reports.

Use GitHub Discussions for:

- Product ideas
- Community questions
- Design principles
- Philosophical or governance discussions
- Proposals that need broader input

## Maintainer Review

Maintainers may ask for changes before merging a pull request.

This is normal. The goal is to keep the project coherent, safe, and aligned with K4M2A’s mission.

## Mission

K4M2A exists to support human awakening, inner transformation, philosophical depth, and collective harmony through open-source tools, learning spaces, and community infrastructure.
CoSeeker (brand) is one part of that larger mission.
